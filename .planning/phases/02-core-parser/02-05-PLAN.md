<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compressed payload bytes → inflateRawSync | Untrusted compressed bytes drive the inflater. Malicious crafted payloads can attempt decompression bombs (small input → huge output) or stream corruption. |
| inflated buffer → parseExplicitLE | Untrusted inflated bytes are then parsed through the Phase 2 Explicit-LE parser; bounds-check + nesting-depth + ByteCursor RangeError handling all apply transitively. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Decompression bomb (resource exhaustion) | inflateRawSync call | mitigate | Pass `{ maxOutputLength: 256 * 1024 * 1024 }` (256 MiB) to `inflateRawSync`. On exceed, Node throws `RangeError` with `code === 'ERR_BUFFER_TOO_LARGE'` (or analogous); catch + convert to `DicomParseError(INVALID_FILE_META, 'Inflated output exceeds 256 MiB cap')`. Test programmatically: build a payload that inflates to ~257 MiB (compressed should be tiny — the classic deflate bomb shape: long runs of identical bytes), verify the typed throw. Use a smaller cap (e.g., 1 KiB) in the test for tractability — the cap is a Phase-2-internal constant, exposed via test-only override. |
| T-02-05-02 | Stream corruption | inflateRawSync call | mitigate | Wrap `inflateRawSync` in try/catch. Any thrown Error → re-throw as `DicomParseError(INVALID_FILE_META, 'Failed to inflate Deflated TS payload: <message>', fileMetaEnd, snippet)`. Test: pass random bytes as the deflate payload → assert typed throw, not a raw zlib error. |
| T-02-05-03 | Buffer over-read inside inflated payload | parseExplicitLE on inflated buffer | mitigate (transitive) | Plan 02-04 already enforces bounds-check + nesting-depth cap on parseExplicitLE; those mitigations apply transitively to the inflated buffer. |

Document on JSDoc that Deflated-TS Element rawBytes pin the inflated buffer; `copyValues: true` opts out. |
| T-02-05-05 | Position-confusion attack | warning byteOffset values | mitigate | The inner emitter wraps warnings to set `position.deflated = true`. **Step 1 — `src/parser/deflated-le.ts`.** Skeleton:

```typescript
import { Buffer } from "node:buffer";
import { inflateRawSync } from "node:zlib";
import { parseExplicitLE } from "./explicit-le.js";
import { DicomParseError, FATAL_CODES, buildSnippet } from "./errors.js";
import type { DicomParseWarning } from "./warnings.js";
import type { ParseContext } from "./types.js";
import type { Element } from "../dataset/element.js";
import type { Tag } from "../dictionary/types.js";

/** Default decompression-bomb cap — 256 MiB (T-02-05-01). Configurable via parseDeflatedLEWithCap for tests; v1.x may expose this via ParseOptions. */
export const DEFAULT_MAX_INFLATED_BYTES = 256 * 1024 * 1024;

/** Deflated Explicit VR Little Endian dataset parser per CONTEXT.md D-26 + D-27 + REQUIREMENTS TS-04.
 *
 * Uses `zlib.inflateRawSync` (RFC 1951 raw deflate). MUST NOT use `inflateSync` (RFC 1950 zlib-wrapped) — see PROJECT.md Key Decisions and PITFALLS.md §1.4.
 *
 * File Meta is parsed UNCOMPRESSED before this function runs (FM-01). Inflation begins at `datasetStart` (= fileMetaEnd) and runs through end-of-buffer. The inflated bytes are then parsed via `parseExplicitLE`. Element byte-offsets in the resulting Dataset are reported relative to the INFLATED buffer with `position.deflated = true` per D-27.
 *
 * @remarks
 * Element rawBytes from a Deflated TS dataset are subarrays of the inflated buffer; the inflated buffer is held in memory until every Element is GC'd. Pass `{ copyValues: true }` to `parseDicom` to release the inflated buffer immediately after parse.
 *
 * Decompression bomb mitigation: caps inflated output at 256 MiB. On exceed, throws `DicomParseError(INVALID_FILE_META)`.
 *
 * @internal
 */
export function parseDeflatedLE(
  buffer: Buffer,
  datasetStart: number,
  ctx: ParseContext,
  emit: (w: DicomParseWarning) => void,
): { elements: ReadonlyMap<Tag, Element>; endOffset: number } {
  return parseDeflatedLEWithCap(buffer, datasetStart, ctx, emit, DEFAULT_MAX_INFLATED_BYTES);
}

/** Internal — exposed only so the test suite can override the cap to a small value (~1 KiB) for tractable decompression-bomb tests. */
export function parseDeflatedLEWithCap(
  buffer: Buffer,
  datasetStart: number,
  ctx: ParseContext,
  emit: (w: DicomParseWarning) => void,
  maxInflatedBytes: number,
): { elements: ReadonlyMap<Tag, Element>; endOffset: number } {
  const compressed = buffer.subarray(datasetStart);

  let inflated: Buffer;
  try {
    inflated = inflateRawSync(compressed, { maxOutputLength: maxInflatedBytes });
  } catch (err) {
    const errAny = err as { code?: string; message?: string };
    const code = errAny.code;
    const message = errAny.message ?? String(err);
    // Node's decompression-bomb signal — `ERR_BUFFER_TOO_LARGE` (>= Node 16.9). Older Node versions surface this as a different RangeError shape; check by class + message.
    if (code === "ERR_BUFFER_TOO_LARGE" || (err instanceof RangeError && /maxOutputLength|too large/i.test(message))) {
      throw new DicomParseError(
        FATAL_CODES.INVALID_FILE_META,
        `Inflated Deflated TS payload exceeds ${String(maxInflatedBytes)}-byte cap.`,
        datasetStart,
        buildSnippet(buffer, datasetStart),
      );
    }
    throw new DicomParseError(
      FATAL_CODES.INVALID_FILE_META,
      `Failed to inflate Deflated TS payload: ${message}`,
      datasetStart,
      buildSnippet(buffer, datasetStart),
    );
  }

  // Inner ParseContext over the inflated buffer.
  const innerCtx: ParseContext = { ...ctx, buffer: inflated };

  // Inner emit wrapper — tags every emitted warning's position with `deflated: true` (D-27).
  const innerEmit = (w: DicomParseWarning): void => {
    const wrapped: DicomParseWarning = {
      ...w,
      position: { ...w.position, deflated: true },
    };
    emit(wrapped);
  };

  const result = parseExplicitLE(inflated, 0, innerCtx, innerEmit);
  return { elements: result.elements, endOffset: buffer.length };
}
```

export function buildDicom(opts: BuildDicomOptions): Buffer {
  const isDeflated = opts.transferSyntax === "1.2.840.10008.1.2.1.99";

  // 1. Preamble + DICM (unless skipped) + File Meta — same as before.
  const preambleAndFm = buildPreambleAndFileMeta(opts);

  // 2. Dataset elements.
  let datasetBytes: Buffer;
  if (isDeflated) {
    // Encode each element as Explicit VR LE, then deflate the concatenation.
    const explicitLeBytes = Buffer.concat(opts.elements.map((el) =>
      encodeElement(el, "1.2.840.10008.1.2.1")
    ));
    datasetBytes = deflateRawSync(explicitLeBytes);
  } else {
    datasetBytes = Buffer.concat(opts.elements.map((el) => encodeElement(el, opts.transferSyntax)));
  }

  return Buffer.concat([preambleAndFm, datasetBytes]);
}
