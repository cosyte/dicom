Implementation outline:

```ts
#!/usr/bin/env tsx
/**
 * Phase 1 Plan 04 PHI scanner — TEST-09 CI-scan half.
 *
 * Pure Node. Zero runtime deps. Walks committed/staged DICOM fixtures and
 * non-DICOM data files under `test/fixtures/**` and rejects:
 *   1. PN values not matching the synthetic allow-list (scripts/phi-allow-list.txt)
 *   2. DA / DT values within the last 120 years of TODAY
 *
 * SECURITY: All git invocations use execFileSync with array args. No shell.
 *
 * Modes:
 *   --staged                 - scan only files staged in `git diff --cached --name-only`
 *   --allow-fixture <path>   - bypass for one path; rejected if not logged in phi-scan-overrides.md
 *   <path> [<path>...]       - scan specific paths
 *   (no args)                - scan all test/fixtures/** files in the working tree
 *
 * Exit codes: 0 (clean), 1 (hits found), 2 (invocation error).
 *
 * D-15 / D-16 / D-17 / TEST-09.
 */

import { readFileSync, statSync, existsSync, readdirSync } from "node:fs";
import { execFileSync } from "node:child_process";
import { join, resolve, relative, sep, extname } from "node:path";

// 1. Parse argv
//    --staged
//    --allow-fixture <path>
//    positional paths
//    Validate exclusivity: --staged + positional paths is an error (exit 2)
//
// 2. Resolve target file list
//    --staged -> execFileSync("git", ["diff","--cached","--name-only","--diff-filter=AM","-z"]) then filter to test/fixtures/**
//    positional -> use as given (must exist)
//    no args -> walk test/fixtures/** with readdirSync({ withFileTypes: true, recursive: true })
//
// 3. Load allow-list
//    Read scripts/phi-allow-list.txt
//    PN allow-list:    Set of trimmed non-comment lines NOT prefixed with "DATE:"
//    DATE allow-list:  Set of YYYYMMDD strings extracted from "DATE:" lines
//
// 4. Validate --allow-fixture overrides
//    Read phi-scan-overrides.md
//    Confirm every <path> from --allow-fixture <path> appears under a `### <path>` heading
//    Reject (exit 2) if not.
//
// 5. For each target file, dispatch by extension:
//    .dcm  -> scanDicom(path, buffer)
//    .json -> scanText(path, buffer.toString('utf8'), 'json')
//    .txt  -> scanText(path, buffer.toString('utf8'), 'txt')
//    .bin  -> scanDicom(path, buffer) // best-effort
//    For --staged mode: read the staged blob via execFileSync("git", ["show", `:${path}`], { encoding: "buffer" })
//    NOT the working-tree content (developer may have reverted in working tree to hide).
//
// 6. scanDicom:
//    a. Verify offset 128 == "DICM" (otherwise warn but continue as best-effort)
//    b. Walk File Meta group (always Explicit VR LE) to find (0008,0005) charset
//       (default to ASCII if absent for our fixtures)
//    c. Walk dataset using Explicit VR LE (most common for tests).
//       Implicit VR LE handling: hardcode (0010,0010) PN, (0008,0020) DA, (0008,0021) DA, etc. tags.
//       Full list:
//         PN tags:  (0010,0010), (0008,0090), (0008,1048), (0008,1050), (0008,1060), (0008,1070),
//                   (0010,1001), (0010,1005), (0010,1060), (0040,A123)
//         DA tags:  (0008,0020), (0008,0021), (0008,0022), (0008,0023), (0010,0030), (0040,A030)
//         DT tags:  (0008,002A), (0040,A12C), (0040,A13A)
//    d. For each matched element, read its value as a string (ASCII / Latin-1) and check:
//       - PN: not in allow-list AND not starting with any allow-list prefix (e.g., "ANON^*")
//       - DA: parse YYYYMMDD; if year >= (THIS_YEAR - 120) AND not in DATE allow-list -> HIT
//       - DT: parse YYYYMMDDHHMMSS; same date check on first 8 chars
//
// 7. scanText:
//    a. Regex sweep for ISO-style dates: /\b(\d{4})(\d{2})(\d{2})\b/g and /\b\d{4}-\d{2}-\d{2}\b/g
//    b. For each match, year >= (THIS_YEAR - 120) AND not in DATE allow-list -> HIT
//    c. PN sweep: regex for /\b[A-Z][A-Za-z\-']+\^[A-Z][A-Za-z\-']+/g (FAMILY^GIVEN)
//       For each match, allow-list check (prefix match OR exact match)
//
// 8. Report:
//    For each hit, print to stderr in the structured format above (using process.stderr.write,
//    NOT console.log/error if eslint complains; scripts/ allows console.* but stderr direct write
//    is preferable for clean piping).
//    Suppress hits for paths in --allow-fixture (only after the override-log validation step 4).
//    Final line: "[phi-scan] N hits across M file(s)" or "[phi-scan] OK" on clean.
//    Exit 0 if N==0, 1 if N>0.
//
// 9. Performance: the scanner reads each file once; tag-walk is bounded by file size.
//    Test fixtures are small (<= 1 MB each typically), so per-file scan is ms.
```

The executor implements the above. Use `console.log(...)` for normal progress and `process.stderr.write(...)` (or `console.error(...)`) for hits — both allowed in `scripts/**/*.ts` per plan 01-01 task 2's eslint override.

Critical implementation details:
- **DICOM File Meta is ALWAYS Explicit VR LE** regardless of dataset transfer syntax — exploit this. Read group `0002,xxxx` first to find `(0002,0010) Transfer Syntax UID`, then either keep Explicit VR LE (`1.2.840.10008.1.2.1`) or switch to Implicit VR LE (`1.2.840.10008.1.2`). For Phase 1 fixtures we control, just commit fixtures matching one of these.
- **Element header parsing (Explicit VR LE):** group (2 bytes LE), element (2 bytes LE), VR (2 ASCII bytes), then either:
  - SHORT-FORM length (most VRs): 2-byte LE length
  - LONG-FORM length (`OB OW OF OD OL SQ UT UN UC UR`): 2 reserved bytes + 4-byte LE length
- **Element header parsing (Implicit VR LE):** group (2 bytes LE), element (2 bytes LE), 4-byte LE length. VR resolved from a tiny hardcoded subset (just the PN/DA/DT tags above).
- **PN value reading:** ASCII / Latin-1 trim trailing spaces / NULLs. Don't bother with charset extension for the scanner — fixtures we ship use ASCII PN values.
- **DA value reading:** strict YYYYMMDD; if value doesn't match `^\d{8}$`, skip the 120-year check (treat as malformed, not a hit). Phase 2 parser does the lenient handling; the scanner stays strict.
- **120-year threshold:** `const CUTOFF_YEAR = new Date().getFullYear() - 120;` — anything `>=` CUTOFF_YEAR is a hit. So in 2026 the cutoff is 1906; `19000101` passes (allow-listed via DATE: entry above too, belt-and-suspenders).
- **Walk scope (D-15):** When called with no positional args, scope the directory walk to `test/fixtures/**` ONLY — never `vendor/**`, `src/**`, or `.git/**`. This matches D-15 spec verbatim.

For `--staged` mode: invoke `git diff --cached --name-only --diff-filter=AM -z` (NUL-separated) to enumerate staged files via `execFileSync` (array form, no shell), then filter to those matching `test/fixtures/**`. Read the staged BLOB content (NOT working-tree content) via `execFileSync("git", ["show", \`:${path}\`])`. This is critical — the working tree may differ from staged content.
